Provides utilities for configuring Kubernetes client transport settings, specifically for handling TLS certificate verification in local development environments where self-signed certificates or hostname mismatches are common.

## Key Components

### Functions

- **`ApplyInsecureTLSConfig`** - Main function that configures a `rest.Config` to skip TLS certificate verification while preserving client authentication
- **`ApplyInsecureTransport`** - Deprecated alias for backward compatibility

## Usage Example

```go
package main

import (
    "k8s.io/client-go/rest"
    "k8s.io/client-go/tools/clientcmd"
    "your-project/config"
)

func main() {
    // Load kubeconfig
    kubeconfig := clientcmd.NewDefaultClientConfigLoadingRules().GetDefaultFilename()
    restConfig, err := clientcmd.BuildConfigFromFlags("", kubeconfig)
    if err != nil {
        panic(err)
    }
    
    // Apply insecure TLS config for local development
    restConfig = config.ApplyInsecureTLSConfig(restConfig)
    
    // Use the config with kubernetes client
    // The client will skip server certificate verification
    // but still use client certificates for authentication
}
```

This is particularly useful for k3d, WSL2, and other local Kubernetes environments where the API server certificate doesn't match localhost addresses. The function preserves client authentication while bypassing server certificate validation - use only for development, never in production.
