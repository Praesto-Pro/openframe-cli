Windows-specific path handling utility that expands Windows 8.3 short filenames to their full long path names, essential for WSL compatibility when working with Windows file paths.

## Key Components

- **expandShortPath(path string) (string, error)**: Main function that converts Windows short paths (e.g., `RUNNER~1`) to full long paths using the Windows API
- **kernel32**: Lazy-loaded Windows kernel32.dll for system calls
- **procGetLongPathNameW**: Windows API procedure for path name expansion

## Usage Example

```go
// Expand a Windows short path to its full form
shortPath := "C:\\Users\\RUNNER~1\\AppData\\Local"
longPath, err := expandShortPath(shortPath)
if err != nil {
    log.Printf("Failed to expand path: %v", err)
    return
}
// Result: "C:\\Users\\runneradmin\\AppData\\Local"
fmt.Printf("Expanded path: %s\n", longPath)

// Handle empty paths safely
emptyPath, _ := expandShortPath("")
// Returns empty string without error

// Fallback behavior for invalid paths
invalidPath, _ := expandShortPath("C:\\NonExistent\\Path")
// Returns original path if expansion fails
```

This utility is particularly important in containerized environments where WSL needs to interact with Windows file systems, as WSL cannot properly resolve Windows short filename formats. The function provides graceful fallback by returning the original path if expansion fails.
