Central system component that validates required tools are installed and configured before cluster operations can proceed.

## Key Components

- **PrerequisiteChecker**: Main struct that manages a collection of tool requirements and validates their installation status
- **Requirement**: Configuration struct defining each tool's name, command, installation check, and help instructions
- **CheckAll()**: Validates all prerequisites and returns missing tools list
- **GetInstallInstructions()**: Provides installation guidance for missing tools
- **CheckPrerequisites()**: Convenience function that detects CI environments and runs appropriate validation

## Usage Example

```go
// Create a new checker instance
checker := NewPrerequisiteChecker()

// Validate all required tools
allPresent, missing := checker.CheckAll()

if !allPresent {
    // Get installation help for missing tools
    instructions := checker.GetInstallInstructions(missing)
    
    for i, tool := range missing {
        fmt.Printf("Missing: %s\n", tool)
        fmt.Printf("Install: %s\n\n", instructions[i])
    }
}

// Quick validation in CI/CD environments
if err := CheckPrerequisites(); err != nil {
    log.Fatal("Prerequisites check failed:", err)
}
```

The checker validates four essential tools: Docker (with daemon running check), kubectl, k3d, and Helm. Each requirement uses dedicated installer modules that provide platform-specific installation guidance and status verification.
