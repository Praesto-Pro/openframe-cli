Handles automated installation and management of OpenFrame CLI prerequisites including Docker, kubectl, k3d, and Helm with interactive and non-interactive modes.

## Key Components

- **`Installer`** - Main installer struct with prerequisite checker integration
- **`NewInstaller()`** - Creates a new installer instance
- **`InstallMissingPrerequisites()`** - Installs all missing prerequisites with progress tracking
- **`CheckAndInstall()`** - Checks and installs prerequisites with user interaction
- **`CheckAndInstallNonInteractive()`** - Non-interactive installation mode for automation
- **`installTool()`** - Installs individual tools (Docker, kubectl, k3d, Helm)
- **`showManualInstructions()`** - Displays manual installation instructions in table format
- **`showDockerStartInstructions()`** - Shows platform-specific Docker startup instructions

## Usage Example

```go
// Create installer and install missing prerequisites
installer := NewInstaller()
if err := installer.CheckAndInstall(); err != nil {
    log.Fatal("Failed to install prerequisites:", err)
}

// Non-interactive mode for CI/automation
if err := installer.CheckAndInstallNonInteractive(true); err != nil {
    log.Fatal("Automated installation failed:", err)
}

// Install specific missing tools only
if err := installer.InstallMissingPrerequisites(); err != nil {
    log.Fatal("Installation failed:", err)
}
```

The installer provides a two-phase approach: first installing missing tools, then handling Docker startup separately. It supports both interactive prompts for user confirmation and automated non-interactive modes for CI environments. Progress is displayed using spinners, and detailed manual instructions are provided when automatic installation fails or is declined.
