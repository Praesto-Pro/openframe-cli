Provides Windows-specific path expansion functionality to convert 8.3 short filenames to their full long path names, ensuring compatibility with WSL environments that don't recognize Windows short paths.

## Key Components

### Functions
- **`expandShortPath(path string) (string, error)`** - Main function that expands Windows short paths to long paths using the Windows API

### Windows API Integration
- **`kernel32`** - Lazy-loaded Windows kernel32.dll library
- **`procGetLongPathNameW`** - Windows API procedure for getting long path names

## Usage Example

```go
// Example of expanding a Windows 8.3 short path
shortPath := "C:\\Users\\RUNNER~1\\Documents"
longPath, err := expandShortPath(shortPath)
if err != nil {
    log.Printf("Failed to expand path: %v", err)
    return
}

// Result: "C:\\Users\\runneradmin\\Documents"
fmt.Printf("Expanded path: %s\n", longPath)

// Function gracefully handles edge cases
emptyPath, _ := expandShortPath("")  // Returns empty string
normalPath, _ := expandShortPath("C:\\Program Files")  // Returns unchanged if already long
```

The function uses the Windows `GetLongPathNameW` API with a two-call pattern: first to determine the required buffer size, then to retrieve the actual long path. It includes error handling and fallback behavior, returning the original path if expansion fails, making it safe to use in production environments.
