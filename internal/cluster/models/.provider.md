Defines the core interfaces for the cluster provider architecture, establishing contracts for cluster lifecycle management and multi-provider support.

## Key Components

### ClusterProvider Interface
Core contract for cluster provider implementations with methods for:
- `Create()` - Creates new clusters with given configuration
- `Delete()` - Removes clusters by name with optional force flag
- `Start()` - Starts stopped clusters
- `List()` - Returns all managed clusters
- `Status()` - Gets detailed cluster status information
- `DetectType()` - Identifies cluster type for given cluster name
- `GetKubeconfig()` - Retrieves kubeconfig for cluster access

### ClusterService Interface
Business logic layer for cluster operations:
- `CreateCluster()` - Creates clusters and returns `*rest.Config`
- `DeleteCluster()` - Removes clusters with type-specific handling
- `StartCluster()` - Starts clusters using appropriate provider
- `ListClusters()` - Aggregates clusters across all providers
- `GetClusterStatus()` - Retrieves status with provider abstraction
- `DetectClusterType()` - Determines cluster type
- `GetRestConfig()` - Gets Kubernetes client configuration

### ProviderRegistry Interface
Registry pattern for managing multiple cluster providers:
- `RegisterProvider()` - Associates providers with cluster types
- `GetProvider()` - Retrieves provider for specific cluster type
- `GetAllProviders()` - Returns complete provider mapping
