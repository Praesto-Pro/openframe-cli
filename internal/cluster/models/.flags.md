This file defines command-line flags and validation logic for the OpenFrame CLI cluster management commands.

## Key Components

**Flag Structures:**
- `GlobalFlags` - Type alias for common flags shared across commands
- `CreateFlags` - Flags for cluster creation (type, node count, K8s version, skip wizard)
- `ListFlags` - Flags for listing clusters (quiet mode)
- `StatusFlags` - Flags for cluster status (detailed view, skip apps)
- `DeleteFlags` - Flags for cluster deletion (force confirmation skip)
- `CleanupFlags` - Flags for cleanup operations (aggressive cleanup mode)

**Flag Setup Functions:**
- `AddGlobalFlags()` - Adds common flags to any command
- `Add*Flags()` functions - Add command-specific flags to cobra commands

**Validation Functions:**
- `ValidateClusterName()` - Validates Kubernetes DNS-1123 subdomain naming rules
- `Validate*Flags()` functions - Validate flag combinations for each command

## Usage Example

```go
// Setting up create command with flags
var createFlags CreateFlags
createCmd := &cobra.Command{
    Use: "create",
    RunE: func(cmd *cobra.Command, args []string) error {
        // Validate cluster name from args
        if err := ValidateClusterName(args[0]); err != nil {
            return err
        }
        
        // Validate flag combinations
        if err := ValidateCreateFlags(&createFlags); err != nil {
            return err
        }
        
        // Use flags in cluster creation logic
        return createCluster(args[0], &createFlags)
    },
}

// Add flags to command
AddGlobalFlags(createCmd, &createFlags.GlobalFlags)
AddCreateFlags(createCmd, &createFlags)
```

The validation ensures cluster names follow Kubernetes naming conventions (alphanumeric + hyphens, 1-63 characters, start/end with alphanumeric) and validates flag combinations for each command type.
